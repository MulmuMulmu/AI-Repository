# OCR 전처리 고도화 벤치마크 보고서

> 작성일: 2026-04-17  
> 환경: Python 3.9 / PaddleOCR v5 (korean) / macOS ARM64

---

## 1. 배경 및 목적

현재 영수증 OCR 파이프라인의 인식 성능이 부족하다는 PM 피드백에 따라, 다음 두 가지를 검증하였다.

1. **PIL 기반 합성 영수증 데이터**를 생성하여 OCR 정확도를 정량적으로 측정할 수 있는가?
2. **전처리 파이프라인을 고도화**(업스케일링, 노이즈 제거, 적응형 이진화 등)하면 OCR 성능이 개선되는가?

---

## 2. 실험 설계

### 2.1 합성 데이터 생성

PIL 라이브러리로 실제 마트 영수증과 동일한 형태의 합성 이미지를 생성하였다.

- **상품 카탈로그**: 한국 마트 식품 28종 (정육, 유제품, 채소/과일, 해산물, 가공식품 등)
- **영수증 구성**: 매장명, 구매일시, POS번호, 상품 목록(4~10개), 합계
- **폰트**: AppleSDGothicNeo (시스템 한글 폰트)
- **Ground Truth**: 각 이미지에 대응하는 JSON (상품명, 카테고리, 가격)

### 2.2 열화 단계 (Degradation Levels)

동일한 깨끗한 합성 영수증에 4단계의 열화를 적용하여 실제 촬영 환경을 시뮬레이션하였다.

| 단계 | 설명 | 시뮬레이션 대상 |
|------|------|----------------|
| **clean** | 원본 그대로 | 스캐너, 고품질 촬영 |
| **mild** | 약한 노이즈(σ=8) + 밝기 변동(±10%) | 양호한 조건의 스마트폰 촬영 |
| **medium** | 노이즈(σ=18) + 블러(r=0.8) + 회전(±2.5°) + 조명 그라데이션 | 일반적인 스마트폰 촬영 |
| **severe** | 저해상도(45%↓) + 강한 노이즈(σ=30) + 블러(r=1.5) + 회전(±4°) + 대비 저하 | 열악한 조건 (흔들림, 먼 거리, 어두운 환경) |

### 2.3 비교 대상 전처리 방식

| 방식 | 처리 내용 |
|------|----------|
| **RAW** | 전처리 없음 — 원본 이미지 그대로 PaddleOCR에 전달 |
| **OLD** | Grayscale → AutoContrast → Contrast Enhance(×1.5) |
| **NEW (고도화 시도)** | Grayscale → 업스케일(단변 800px) → MedianFilter 디노이즈 → 적응형 이진화(로컬 평균 기반) → Sharpen |

### 2.4 평가 지표

- **상품명 Recall**: Ground truth 상품명 중 OCR에서 인식된 비율 (SequenceMatcher ≥ 0.5 기준)
- **문자 정확도 (Char Accuracy)**: 매칭된 상품명의 글자 수준 정확도 (1 − 편집거리/참조길이)

## 3. 벤치마크 결과

### 3.1 상품명 인식률 (Recall)

> 합성 영수증 5건 × 4단계 열화 = 총 20건 테스트

| 열화 단계 | RAW (전처리 없음) | OLD (기본 전처리) | NEW (고도화 시도) |
|:---------:|:-:|:-:|:-:|
| clean | 97.8% | **100.0%** | 100.0% |
| mild | **100.0%** | 100.0% | 100.0% |
| medium | **100.0%** | 97.1% | 97.1% |
| severe | **84.4%** | 83.9% | 52.9% |
| **전체 평균** | **95.6%** | **95.3%** | 87.5% |

### 3.2 문자 정확도 (Char Accuracy)

| 열화 단계 | RAW | OLD | NEW |
|:---------:|:-:|:-:|:-:|
| clean | 41% | **56%** | 56% |
| mild | **52%** | 50% | 50% |
| medium | **54%** | 50% | 50% |
| severe | **62%** | 59% | 41% |

### 3.3 전처리 소요 시간

| 열화 단계 | RAW | OLD | NEW |
|:---------:|:-:|:-:|:-:|
| clean | 0ms | 4ms | 36ms |
| mild | 0ms | 10ms | 12ms |
| medium | 0ms | 10ms | 16ms |
| severe | 0ms | 9ms | 20ms |

## 4. 분석

### 4.1 전처리 없음(RAW) vs 기본 전처리(OLD)

- Recall 차이는 **-0.3%p**(95.6% → 95.3%)로 거의 없음
- 다만 clean 이미지에서 **문자 정확도가 41% → 56%로 +15%p 향상**
- PaddleOCR v5의 딥러닝 모델이 내부적으로 이미 전처리(정규화, 대비 보정 등)를 수행하기 때문에, 외부 전처리의 recall 개선 효과는 미미함
- 문자 정확도 개선 효과가 있으므로 **기본 전처리(OLD)를 유지하는 것이 합리적**

### 4.2 기본 전처리(OLD) vs 고도화 시도(NEW)

- **severe 열화에서 recall이 83.9% → 52.9%로 급격히 하락** (-31.0%p)
- 원인 분석:
  - **업스케일링의 역효과**: 저해상도(45%) 이미지를 800px로 강제 업스케일하면 노이즈와 블러가 동시에 증폭되어, PaddleOCR의 텍스트 검출(detection)이 오히려 혼란
  - **적응형 이진화의 정보 손실**: PaddleOCR은 딥러닝 기반이므로 자연스러운 grayscale 입력에서 최적 성능을 발휘하며, 이진화된 입력은 미세한 텍스처 정보를 소실시킴
  - **디노이즈(MedianFilter)의 텍스트 훼손**: 중앙값 필터가 텍스트 경계를 흐리게 만들어 글자 인식 정확도 하락

### 4.3 핵심 결론

> **PaddleOCR v5와 같은 최신 딥러닝 OCR 엔진에서는 과도한 전처리가 오히려 성능을 악화시킨다.**

기존 기본 전처리(Grayscale + AutoContrast + Contrast 1.5)가 현 시점에서 최적의 전처리 파이프라인이며, 이를 넘어서는 고도화는 역효과를 초래한다.

## 5. 합성 데이터 활용 가치

전처리 튜닝에는 한계가 있지만, 합성 데이터 자체는 다음 용도로 활용 가능하다.

| 용도 | 설명 |
|------|------|
| **OCR 정확도 벤치마크** | 코드 변경 시 회귀(regression) 여부를 정량적으로 검증 |
| **후처리 규칙 개발** | OCR 오독 패턴을 파악하고 텍스트 보정 규칙 작성에 활용 |
| **카테고리 분류 검증** | 상품명 → 8개 카테고리 분류 로직의 정확도 측정 |
| **Qwen 보정 효과 측정** | Qwen 후처리 전/후 정확도를 동일 데이터셋으로 비교 |

## 6. 성능 개선을 위한 권장 방향

현재 전처리는 **기본(OLD) 방식을 유지**하되, 실질적 성능 개선은 다음 방향을 권장한다.

### 단기 (즉시 적용 가능)

| 방안 | 예상 효과 | 난이도 |
|------|----------|--------|
| **촬영 가이드 UI** — 사용자에게 적정 거리·조명·초점 안내 | severe 케이스 자체를 감소 | 낮음 |
| **Qwen 후처리 활성화** — OCR 오독을 LLM으로 보정 | 문자 정확도 향상 | 중간 |
| **상품명 퍼지 매칭 강화** — ingredients.json 기반 유사도 매칭 임계값 조정 | recall 향상 | 낮음 |

### 중기 (1~2주)

| 방안 | 예상 효과 | 난이도 |
|------|----------|--------|
| **PaddleOCR 한국어 fine-tuning** — 실제 영수증 데이터로 recognition 모델 미세조정 | 문자 정확도 대폭 향상 | 높음 |
| **후처리 사전(dictionary) 구축** — 흔한 OCR 오류 패턴 DB화 (예: 우유→우뮤 보정) | 문자 정확도 향상 | 중간 |

### 장기

| 방안 | 예상 효과 | 난이도 |
|------|----------|--------|
| **실제 영수증 라벨링 데이터셋 구축** — 수집된 영수증 사진에 ground truth 라벨 부착 | 전체 파이프라인 품질 측정 기반 확보 | 높음 |
| **멀티모달 OCR (Vision LLM)** — 이미지 전체를 LLM에 직접 입력하여 텍스트 추출 | severe 케이스 대폭 개선 가능 | 높음 |
